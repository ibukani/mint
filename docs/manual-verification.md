# 手動検証手順 (Manual Verification)

Mint アプリケーションのE2Eテスト環境が構築されるまでの間、リリース前や大幅な機能改修後には以下の手動検証を実施してください。

## 対象プラットフォームと画面サイズ

Mint は Tauri のデスクトップ専用アプリです。スマートフォン向けのレイアウトやタッチ操作は手動検証の対象外とし、次のデスクトップ条件を確認します。

- メイン設定画面: 通常サイズ（1280×900前後）、既定サイズ（900×650）、最小サイズ（680×520）
- オーバーレイ: 時計（420×168）、カレンダー（420×384）、ゲームランチャー（760×520）、クイックキャプチャー（680×500）、予定編集（420×640）、ファイルシェル（折りたたみ時 32×96、展開時 360×520）
- ダーク/ライトテーマ、キーボード操作、ウィンドウのドラッグ・最小化・クローズ

## 1. アプリケーション起動
1. `npm run tauri -- dev` を実行。
2. アプリケーションのウィンドウが正常に表示されることを確認。
3. デベロッパーツールを開き、コンソールにエラーが出ていないことを確認。

## 2. 機能管理ダッシュボードの設定保存と永続化
1. 機能管理ダッシュボード（設定画面）で「テーマ」を切り替える（ダーク / ライト / システム）。システムを選んだ場合はOSの外観設定変更に追従することを確認する。
2. アプリを完全に終了し、再起動。
3. 変更したテーマが維持されていることを確認（`settings.json` への永続化テスト）。
4. `Ctrl+K`（macOSは `Command+K`）で設定検索を開き、検索語を入力して「設定検索をクリア」で一覧へ戻れることを確認する。設定値の入力欄にフォーカス中は同ショートカットが入力操作を奪わないことも確認する。

## 3. ショートカットとトレイメニュー
1. 時計機能のショートカットキー（デフォルト等）を変更し保存。
2. 変更したショートカットキーを入力し、時計オーバーレイが表示・非表示されることを確認。
3. トレイアイコンを右クリックし、メニューから「ダッシュボードを開く」「終了」などの操作が動作することを確認。

## 4. ウィンドウとオーバーレイ
1. 時計オーバーレイが指定秒数後に自動非表示になるか確認。
2. Discord風の「×ボタンで最小化（トレイへ）」が機能し、プロセスがキルされないことを確認。

## 5. Voice to Text
1. Voice to Text のダッシュボード設定でAPIキー、Base URL、モデル名、言語、ショートカットを入力し保存。
2. APIキーがOSのキーリングに保存され、その他の設定がファイルに保存されることを確認（エラーが出ないこと）。
3. 設定したグローバルショートカットを別タブまたは非表示状態から押し、メイン画面が音声入力へ切り替わって音声ファイル欄へフォーカスすることを確認する。すでに音声入力を表示中の場合も同じ欄へ戻ることを確認する。
4. 機能を有効にして音声ファイルパスを入力し、ファイル欄のEnter、「文字起こしを実行」、`Ctrl+Enter`（macOSは `Command+Enter`）のいずれでも文字起こし結果が表示されることを確認する。「マイク録音」で権限を許可し、録音停止後に自動で文字起こしされること、録音中に「破棄」を選ぶと送信されずにマイクが解放されることも確認する。
5. 対応する音声ファイル（WAV / MP3 / M4A / AAC / FLAC / OGG / WebM）を文字起こしワークベンチへドラッグし、ドロップ中の案内が表示され、ドロップ後にファイル欄へパスが入り、既存の結果が消えることを確認する。複数ファイルや非対応形式では現在のパスが保持され、次の操作が分かる警告が表示されることも確認する。
6. 処理中に音声ファイル、APIキー、Base URL、モデル、言語、または有効状態を変更し、変更前のリクエストが完了しても古い結果が復活しないことを確認する。
7. APIキー未設定、存在しない音声ファイル、マイク権限拒否、認証エラー、ファイルサイズ超過、利用上限、API停止、タイムアウト時に、原因と次の操作が分かるエラーが表示されることを確認する。通信失敗時は、直前のファイルまたは録音を「再試行」で再送できることも確認する。
8. Voice to Text を無効にした場合、設定したショートカットキーがOSのグローバルショートカットから解除されることを確認。

## 6. カレンダーとGoogle Calendar連携
1. カレンダー設定を開いた直後に、接続確認中の表示を経てから未接続または接続済みの状態が表示され、確認中に未接続画面が一瞬表示されないことを確認する。
2. Googleアカウントを接続し、ブラウザ認証の待機中もMintのウィンドウ移動・設定画面のスクロール・別機能への切り替えが固まらないことを確認する。認証のキャンセルと時間切れでは再操作方法を含むエラーが表示されることも確認する。
3. 接続後に予定表ごとの表示切り替えと、編集可能な予定表だけを新規予定の保存先に選べることを確認する。表示できる予定表がない場合は空状態と再確認操作が表示されることを確認する。
4. 「今すぐ同期」の実行中に進捗が明示され、連打できないことを確認する。カレンダーオーバーレイ表示による同期と同時になった場合も、操作が順番に完了して予定の重複や消失がないことを確認する。オーバーレイの同期失敗時は原因と「再同期」が表示され、通信復旧後に再試行できることも確認する。
5. オフライン、認証切れ、APIエラーから復帰後に再同期でき、更新件数または最新状態の完了メッセージと最終同期日時が表示されることを確認する。
6. 未同期の変更がある場合は件数と必要な操作が表示され、接続解除が無効になることを確認する。同期後は接続解除でき、Google由来の予定と接続状態が消えることを確認する。
7. メイン設定画面の1280×900、900×650、680×520で、ダーク/ライトテーマともにアカウント、予定表一覧、保存先、状態表示、操作ボタンが横にはみ出さないことを確認する。
8. 予定の編集中に閉じる操作またはEscapeを入力すると、未保存の内容を破棄する確認ダイアログが開くことを確認する。初期フォーカスはキャンセル、Tabはダイアログ内を循環し、Escapeまたはキャンセル後は入力内容と元のフォーカスが維持されることを確認する。保存処理中は閉じる操作が無効になることも確認する。
9. 予定詳細の削除では確認前に削除されず、キャンセル後は削除ボタンへフォーカスが戻ることを確認する。削除失敗時はダイアログ内のエラーから再試行でき、Google由来の予定ではGoogle Calendarにも反映される旨が表示されることを確認する。

## 7. ゲームランチャー
1. `Alt+1` でゲームランチャーを開き、Steam、Epic Games、Riot Gamesで検出したゲームが表示されることを確認する。
2. タイトル、ストア名、タイトルの略称で検索でき、「検索をクリア」で一覧へ戻れること、上下キー・Home・Endで選択、Enterで起動できることを確認する。アクションへTab移動した後も `Ctrl+F`（macOSは `Command+F`）または `/` で検索へ戻れることを確認する。
3. ゲームをお気に入りへ追加して先頭へ並び替わった後も、一覧の選択と右側プレビューが同じゲームを保持することを確認する。起動履歴による並び替えでも同様に確認する。
4. 検索中のEscapeは検索だけを解除し、もう一度Escapeを押すと閉じることを確認する。ショートカットの `Alt+1` を再入力した場合は検索状態に関係なく閉じることも確認する。
5. 760×520と620×520のダーク/ライトテーマで横溢れがなく、通常幅ではプレビュー、狭幅では一覧が欠けずに表示されることを確認する。起動失敗、管理画面を開く失敗、再スキャンのエラーも画面内へ表示されることを確認する。

## 8. ファイルシェル
1. Windows Explorer から画面端のハンドルへ複数のファイルとフォルダをドロップし、1つのスタックとして追加されることを確認。
2. `Alt+3` と画面端ハンドルの両方で展開・折りたたみでき、設定した左右の端へ表示されることを確認。
3. シェルフから Explorer へ取り出すと通常はコピー、Shift を押しながら取り出すと移動になり、ドロップ成功時だけ棚から外れることを確認。ドラッグをキャンセルした場合は棚に残ることも確認。
4. 取り出し・削除後の「元に戻す」と `Ctrl+Z`（macOSは `Command+Z`）が10秒以内に機能し、アプリ再起動後も残りの棚項目が維持されることを確認。
5. 画像（PNG/JPEG/GIF/WebP、25MB以下）、URL、文章を `Ctrl+V` で追加し、開く・コピー・Explorerで表示が機能することを確認。
6. 元ファイルをExplorer側で削除した場合に「見つかりません」と表示され、ファイルシェルを無効にすると `Alt+3` が解除されてウィンドウが消えることを確認。
7. クリップボード履歴が初期状態では無効であることを確認。有効化前からクリップボードにある文章は追加されず、有効化後に新しくコピーした64KB以下の文章とURLだけが「履歴」として追加されることを確認。
8. 同じ文章をもう一度コピーすると重複せず先頭へ移動し、設定した保存件数を超えても手動追加した項目は残ることを確認。「クリップボード履歴だけを消去」とUndoも確認。
9. `Ctrl+F`（macOSは `Command+F`）または `/` で棚の検索へ移動し、名前・パス・URL・文章を絞り込めることを確認。上下キー・Home・Endで候補を移動し、Enterで開く、Spaceで選択、`Ctrl+C`（macOSは `Command+C`）でコピーできることを確認する。複数選択時は各項目が改行区切りでコピーされることも確認する。Escapeは検索、選択、棚の順に解除することも確認する。
10. 360×520のダーク/ライトテーマで、検索中と選択アクション表示中も項目が横に欠けず、キーボードカーソルが一覧内へ自動スクロールされることを確認する。ブラウザ確認では `http://localhost:1420/?label=fileShelf&mockClipboardHistory=https%3A%2F%2Fexample.com` を開き、履歴表示と履歴だけの消去も確認する。

## 9. クイックキャプチャー
1. `Alt+2` でオーバーレイを開き、本文とタグを入力して350ms程度で「保存済み」になることを確認する。「コピー」で本文を他アプリへ渡せ、閉じて再度開いた場合も下書きが復元されることを確認する。
2. `Ctrl+Enter`（macOSは `Command+Enter`）または「メモに保存」で下書きを保存済みメモへ変換し、本文欄が空の新しい下書きへ戻ることを確認する。
3. `Ctrl+F`（macOSは `Command+F`）または本文以外で `/` を押し、保存済みメモ検索へ移動することを確認する。上下キー・Home・Endで候補を移動し、Enterで選択したメモを開けることを確認する。
4. 検索欄にフォーカス中のEscapeは、まず絞り込みを解除（絞り込みがなければ検索欄を抜ける）し、もう一度Escapeを押すとオーバーレイが閉じることを確認する。
5. 680×500と600×500のダーク/ライトテーマで、編集欄、タグ、保存操作、検索、メモ一覧が欠けず、キーボード選択中のメモがスクロール領域内に表示されることを確認する。
6. 保存失敗時に入力内容と現在のメモが失われず、「再試行」から保存を再実行できることを確認する。
7. メモ削除とバックアップ復元でMint内の確認ダイアログが表示され、初期フォーカス、Tab循環、Escape、背景クリック、処理中の閉鎖防止が機能することを確認する。復元ではファイル選択後に現在の下書きとメモ件数が示され、失敗後もダイアログ内から再試行できることを確認する。

## 10. アップデーター
1. `npm run dev` を実行し、`http://localhost:1420/?mockUpdate=available` を開く。
2. 一般設定の「アップデート」に、現在・更新後のバージョン、公開日、リリースノートが表示されることを確認。
3. 「ダウンロードして再起動」を押し、ダウンロード進捗、インストール中、再起動中の順に表示が変わることを確認。
4. `http://localhost:1420/` では「更新を確認」を押すと「最新バージョンです」と表示されることを確認。
5. 実リリース前にはテスト用の旧バージョンをインストールし、公開済みの署名付き Release へ更新できることを対象OSで確認。

## 11. プラットフォーム固有
- Windows実機にて、アプリの多重起動が防止されているか（Single Instance）。
- トレイアイコンが Windows 11 のタスクバーで正しく表示されるか。
- Windowsの自動起動は `npm run tauri -- build` で作成したリリース版をインストールして確認する。`npm run tauri -- dev` は `localhost:1420` のViteサーバーを使うため、自動起動には登録されない。
