# 手動検証手順 (Manual Verification)

Mint アプリケーションのE2Eテスト環境が構築されるまでの間、リリース前や大幅な機能改修後には以下の手動検証を実施してください。

## 1. アプリケーション起動
1. `npm run tauri -- dev` を実行。
2. アプリケーションのウィンドウが正常に表示されることを確認。
3. デベロッパーツールを開き、コンソールにエラーが出ていないことを確認。

## 2. 設定保存と永続化
1. 設定画面で「テーマ」を切り替える（Dark / Light）。
2. アプリを完全に終了し、再起動。
3. 変更したテーマが維持されていることを確認（`settings.json` への永続化テスト）。

## 3. ショートカットとトレイメニュー
1. 時計機能のショートカットキー（デフォルト等）を変更し保存。
2. 変更したショートカットキーを入力し、時計オーバーレイが表示・非表示されることを確認。
3. トレイアイコンを右クリックし、メニューから「設定を開く」「終了」などの操作が動作することを確認。

## 4. ウィンドウとオーバーレイ
1. 時計オーバーレイが指定秒数後に自動非表示になるか確認。
2. Discord風の「×ボタンで最小化（トレイへ）」が機能し、プロセスがキルされないことを確認。

## 5. プレースホルダー機能 (Voice to Text 等)
1. Voice to Text 設定でAPIキーやショートカットを入力し保存。
2. これらがOSのキーリング・ファイルに保存されるか確認（エラーが出ないこと）。
3. 実際にショートカットを押しても、文字起こしが走らずエラーにもならない（未実装状態として安全に無視される）ことを確認。

## 6. プラットフォーム固有
- Windows実機にて、アプリの多重起動が防止されているか（Single Instance）。
- トレイアイコンが Windows 11 のタスクバーで正しく表示されるか。
