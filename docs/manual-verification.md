# 手動検証手順 (Manual Verification)

Mint アプリケーションのE2Eテスト環境が構築されるまでの間、リリース前や大幅な機能改修後には以下の手動検証を実施してください。

## 対象プラットフォームと画面サイズ

Mint は Tauri のデスクトップ専用アプリです。スマートフォン向けのレイアウトやタッチ操作は手動検証の対象外とし、次のデスクトップ条件を確認します。

- メイン設定画面: 通常サイズ（1280×900前後）、既定サイズ（900×650）、最小サイズ（680×520）
- オーバーレイ: 時計（420×168）、カレンダー（420×384）、ゲームランチャー（760×520）、予定編集（420×640）、ファイルシェル（折りたたみ時 32×96、展開時 360×520）
- ダーク/ライトテーマ、キーボード操作、ウィンドウのドラッグ・最小化・クローズ

## 1. アプリケーション起動
1. `npm run tauri -- dev` を実行。
2. アプリケーションのウィンドウが正常に表示されることを確認。
3. デベロッパーツールを開き、コンソールにエラーが出ていないことを確認。

## 2. 機能管理ダッシュボードの設定保存と永続化
1. 機能管理ダッシュボード（設定画面）で「テーマ」を切り替える（Dark / Light）。
2. アプリを完全に終了し、再起動。
3. 変更したテーマが維持されていることを確認（`settings.json` への永続化テスト）。

## 3. ショートカットとトレイメニュー
1. 時計機能のショートカットキー（デフォルト等）を変更し保存。
2. 変更したショートカットキーを入力し、時計オーバーレイが表示・非表示されることを確認。
3. トレイアイコンを右クリックし、メニューから「ダッシュボードを開く」「終了」などの操作が動作することを確認。

## 4. ウィンドウとオーバーレイ
1. 時計オーバーレイが指定秒数後に自動非表示になるか確認。
2. Discord風の「×ボタンで最小化（トレイへ）」が機能し、プロセスがキルされないことを確認。

## 5. Voice to Text
1. Voice to Text のダッシュボード設定でAPIキー、Base URL、モデル名、言語、ショートカットを入力し保存。
2. APIキーがOSのキーリングに保存され、その他の設定がファイルに保存されることを確認（エラーが出ないこと）。
3. 機能を有効にして音声ファイルパスを入力し、「文字起こしを実行」で文字起こし結果が表示されることを確認。
4. APIキー未設定、無効な音声ファイルパス、APIエラー時に画面上へエラーが表示されることを確認。
5. Voice to Text を無効にした場合、設定したショートカットキーがOSのグローバルショートカットから解除されることを確認。

## 6. ファイルシェル
1. Windows Explorer から画面端のハンドルへ複数のファイルとフォルダをドロップし、1つのスタックとして追加されることを確認。
2. `Alt+3` と画面端ハンドルの両方で展開・折りたたみでき、設定した左右の端へ表示されることを確認。
3. シェルフから Explorer へ取り出すと通常はコピー、Shift を押しながら取り出すと移動になり、ドロップ成功時だけ棚から外れることを確認。ドラッグをキャンセルした場合は棚に残ることも確認。
4. 取り出し・削除後の「元に戻す」が10秒以内に機能し、アプリ再起動後も残りの棚項目が維持されることを確認。
5. 画像（PNG/JPEG/GIF/WebP、25MB以下）、URL、文章を `Ctrl+V` で追加し、開く・コピー・Explorerで表示が機能することを確認。
6. 元ファイルをExplorer側で削除した場合に「見つかりません」と表示され、ファイルシェルを無効にすると `Alt+3` が解除されてウィンドウが消えることを確認。
7. クリップボード履歴が初期状態では無効であることを確認。有効化前からクリップボードにある文章は追加されず、有効化後に新しくコピーした64KB以下の文章とURLだけが「履歴」として追加されることを確認。
8. 同じ文章をもう一度コピーすると重複せず先頭へ移動し、設定した保存件数を超えても手動追加した項目は残ることを確認。「クリップボード履歴だけを消去」とUndoも確認。
9. ブラウザ確認では `http://localhost:1420/?label=fileShelf&mockClipboardHistory=https%3A%2F%2Fexample.com` を開き、履歴表示と履歴だけの消去を確認。

## 7. アップデーター
1. `npm run dev` を実行し、`http://localhost:1420/?mockUpdate=available` を開く。
2. 一般設定の「アップデート」に、現在・更新後のバージョン、公開日、リリースノートが表示されることを確認。
3. 「ダウンロードして再起動」を押し、ダウンロード進捗、インストール中、再起動中の順に表示が変わることを確認。
4. `http://localhost:1420/` では「更新を確認」を押すと「最新バージョンです」と表示されることを確認。
5. 実リリース前にはテスト用の旧バージョンをインストールし、公開済みの署名付き Release へ更新できることを対象OSで確認。

## 8. プラットフォーム固有
- Windows実機にて、アプリの多重起動が防止されているか（Single Instance）。
- トレイアイコンが Windows 11 のタスクバーで正しく表示されるか。
- Windowsの自動起動は `npm run tauri -- build` で作成したリリース版をインストールして確認する。`npm run tauri -- dev` は `localhost:1420` のViteサーバーを使うため、自動起動には登録されない。
