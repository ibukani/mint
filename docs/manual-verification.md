# 手動検証手順 (Manual Verification)

Mint アプリケーションのE2Eテスト環境が構築されるまでの間、リリース前や大幅な機能改修後には以下の手動検証を実施してください。

## 対象プラットフォームと画面サイズ

Mint は Tauri のデスクトップ専用アプリです。スマートフォン向けのレイアウトやタッチ操作は手動検証の対象外とし、次のデスクトップ条件を確認します。

- メイン設定画面: 通常サイズ（1280×900前後）、既定サイズ（900×650）、最小サイズ（680×520）
- オーバーレイ: 時計（420×168）、カレンダー（420×384）、ゲームランチャー（760×520）、クイックキャプチャー（760×540）、予定編集（420×640）、ファイルシェル（折りたたみ時 32×96、展開時 360×520）
- ダーク/ライトテーマ、キーボード操作、ウィンドウのドラッグ・最小化・クローズ

## 1. アプリケーション起動
1. `npm run tauri -- dev` を実行。
2. アプリケーションのウィンドウが正常に表示されることを確認。
3. デベロッパーツールを開き、コンソールにエラーが出ていないことを確認。

## 2. 機能管理ダッシュボードの設定保存と永続化
1. 機能管理ダッシュボード（設定画面）で「テーマ」を切り替える（ダーク / ライト / システム）。システムを選んだ場合はOSの外観設定変更に追従することを確認する。
2. アプリを完全に終了し、再起動。
3. 変更したテーマが維持されていることを確認（`settings.json` への永続化テスト）。
4. `Ctrl+K`（macOSは `Command+K`）でクイックランチャーを開き、「APIキー」「時計サイズ」などの設定項目を検索して選択すると、該当する入力欄へフォーカスとスクロールが移ることを確認する。「時計を開く」「カレンダーを開く」などの操作を検索して選択した場合は、対応するオーバーレイが表示され、「今日の予定を追加」では予定入力画面が今日の日付で開くことも確認する。無効化中の操作には実行前に「無効」と理由が表示され、選択してもランチャーが閉じず、「詳細設定を開く」で対応する有効化スイッチへ移動でき、設定変更後に再試行できることを確認する。成功した設定・操作は次回ランチャーを開いたとき「最近使った項目」として候補の先頭に表示され、アプリを再起動しても維持されることを確認する。「ライトテーマにする」「ダークテーマにする」「システムテーマにする」を選ぶと、設定画面を移動せずテーマが切り替わることも確認する。実行中の予期しないエラーではランチャー内にエラーが残り、再試行できること、「クイックランチャーの検索をクリア」で一覧へ戻れること、設定値の入力欄にフォーカス中は同ショートカットが入力操作を奪わないことも確認する。

## 3. ショートカットとトレイメニュー
1. 時計機能のショートカットキー（デフォルト等）を変更し保存。
2. 変更したショートカットキーを入力し、時計オーバーレイが表示・非表示されることを確認。
3. トレイアイコンを右クリックし、メニューから「ダッシュボードを開く」「終了」などの操作が動作することを確認。

## 4. ウィンドウとオーバーレイ
1. 時計オーバーレイが指定秒数後に自動非表示になるか確認。
2. Discord風の「×ボタンで最小化（トレイへ）」が機能し、プロセスがキルされないことを確認。
3. メイン設定画面と時計・カレンダー・ゲームランチャー・クイックキャプチャー・予定編集を一度開いて閉じ、2分以上操作しない。各WebViewが破棄された後も、トレイ/設定ショートカットまたは各ショートカットで再生成されて正常に表示されることを確認する。ファイルシェルはエッジハンドル有効時に常駐するため、この確認対象外とする。CPU/RSSの値はOS、WebView、GPU、表示倍率、開発/リリースモードで変わるため、同一PC・同一モードでの前後比較として記録する。

## 5. Voice to Text
1. Voice to Text のダッシュボード設定でAPIキー、Base URL、モデル名、言語、ショートカットを入力し保存する。OpenAIまたはGroqを使う場合は接続先プリセットでBase URLとモデル名をまとめて入力でき、必要に応じてカスタム値へ編集できることも確認する。
2. APIキーがOSのキーリングに保存され、その他の設定がファイルに保存されることを確認（エラーが出ないこと）。
3. 設定したグローバルショートカットを別タブまたは非表示状態から押し、メイン画面が音声入力へ切り替わって音声ファイル欄へフォーカスすることを確認する。すでに音声入力を表示中の場合も同じ欄へ戻ることを確認する。
4. 機能を有効にして音声ファイルパスを入力し、ファイル欄のEnter、「文字起こしを実行」、`Ctrl+Enter`（macOSは `Command+Enter`）のいずれでも文字起こし結果が表示されることを確認する。「マイク録音」で権限を許可し、録音停止後に自動で文字起こしされること、録音中に「破棄」を選ぶと送信されずにマイクが解放されることも確認する。結果の「メモとして保存」を押すと、文字起こし結果が「文字起こし」タグ付きでクイックキャプチャーへ一度だけ保存され、保存後は二重登録できないことを確認する。クイックキャプチャーを開いたまま保存した場合は一覧に新しいメモが反映され、編集中の下書きが置き換わらないことも確認する。
5. 対応する音声ファイル（WAV / MP3 / M4A / AAC / FLAC / OGG / WebM）を文字起こしワークベンチへドラッグし、ドロップ中の案内が表示され、ドロップ後にファイル欄へパスが入り、既存の結果が消えることを確認する。複数ファイルや非対応形式では現在のパスが保持され、次の操作が分かる警告が表示されることも確認する。
6. 処理中に音声ファイル、APIキー、Base URL、モデル、言語、または有効状態を変更し、変更前のリクエストが完了しても古い結果が復活しないことを確認する。
7. APIキー未設定、存在しない音声ファイル、マイク権限拒否、認証エラー、ファイルサイズ超過、利用上限、API停止、タイムアウト時に、原因と次の操作が分かるエラーが表示されることを確認する。通信失敗時は、直前のファイルまたは録音を「再試行」で再送できることも確認する。
8. Voice to Text を無効にした場合、設定したショートカットキーがOSのグローバルショートカットから解除されることを確認。

## 6. カレンダーとGoogle Calendar連携
1. カレンダー設定を開いた直後に、接続確認中の表示を経てから未接続または接続済みの状態が表示され、確認中に未接続画面が一瞬表示されないことを確認する。
2. Googleアカウントを接続し、ブラウザ認証の待機中もMintのウィンドウ移動・設定画面のスクロール・別機能への切り替えが固まらないことを確認する。認証のキャンセルと時間切れでは再操作方法を含むエラーが表示されることも確認する。
3. 接続後に予定表ごとの表示切り替えと、編集可能な予定表だけを新規予定の保存先に選べることを確認する。表示できる予定表がない場合は空状態と再確認操作が表示されることを確認する。月表示の前月／次月ボタンで移動した場合も、同じ日付のグリッドへ選択とフォーカスが戻ることを確認する。「次の予定」を選ぶと予定の月へ移動して詳細を開き、戻るとその月の表示へ戻ることも確認する。
4. 「今すぐ同期」の実行中に進捗が明示され、連打できないことを確認する。カレンダーオーバーレイ表示時の自動同期は、同じ予定表を5分以内に同期済みなら再通信せず、予定表の選択変更または鮮度期限後には同期することを確認する。自動同期と手動同期が同時になった場合も、操作が順番に完了して予定の重複や消失がないことを確認する。オーバーレイの同期失敗時は原因と「再同期」が表示され、通信復旧後に再試行できることも確認する。月表示の「日付」から遠い日付を選ぶと、その月へ移動して選択日へフォーカスが移り、`G` でも日付選択へフォーカスできることを確認する。
5. オフライン、認証切れ、APIエラーから復帰後に再同期でき、更新件数または最新状態の完了メッセージと最終同期日時が表示されることを確認する。ローカル予定の読み込み失敗時は、月表示・日表示のどちらにも「再読み込み」が表示され、復帰できることも確認する。
6. 未同期の変更がある場合は件数と必要な操作が表示され、接続解除が無効になることを確認する。同期後は接続解除でき、Google由来の予定と接続状態が消えることを確認する。
7. メイン設定画面の1280×900、900×650、680×520で、ダーク/ライトテーマともにアカウント、予定表一覧、保存先、状態表示、操作ボタンが横にはみ出さないことを確認する。
8. 予定の編集中に閉じる操作またはEscapeを入力すると、未保存の内容を破棄する確認ダイアログが開くことを確認する。初期フォーカスはキャンセル、Tabはダイアログ内を循環し、Escapeまたはキャンセル後は入力内容と元のフォーカスが維持されることを確認する。保存処理中は閉じる操作が無効になることも確認する。カレンダーを表示したまま予定を保存した場合は、月表示・日表示の一覧と開いている予定詳細が保存内容へ自動更新されることを確認する。予定を別の日付・別月へ移動して保存した場合も、詳細の日時と戻り先の日付が保存内容に追従することを確認する。
9. 予定詳細の「詳細をコピー」または `C` で、タイトル・日時・メモが改行区切りでクリップボードへ入り、成功表示が出ることを確認する。クリップボードが利用できない場合は、予定を失わずエラーが表示されることも確認する。
10. 予定詳細の削除では確認前に削除されず、キャンセル後は削除ボタンへフォーカスが戻ることを確認する。削除失敗時はダイアログ内のエラーから再試行でき、Google由来の予定ではGoogle Calendarにも反映される旨が表示されることを確認する。
11. 日表示を開いたとき、予定があれば最初の予定、空の日は追加ボタン、読み込み失敗時は再読み込みへフォーカスが移ることを確認する。前の日/次の日ボタンまたは←→で日付を連続移動でき、月をまたぐ場合も予定が再読み込みされることを確認する。予定にフォーカスした状態で上下キー・Home・Endで一覧内を移動し、Enterで詳細を開けることを確認する。

## 7. ゲームランチャー
1. `Alt+1` でゲームランチャーを開き、Steam、Epic Games、Riot Gamesで検出したゲームが表示されることを確認する。
2. タイトル、ストア名、タイトルの略称で検索でき、「検索をクリア」で一覧へ戻れること、上下キーで1件、PageUp/PageDownで5件、Home/Endで先頭・末尾へ選択を移動し、Enterで起動できることを確認する。アクションへTab移動した後も `Ctrl+F`（macOSは `Command+F`）または `/` で検索へ戻れることを確認する。
3. ゲームをお気に入りへ追加して先頭へ並び替わった後も、一覧の選択と右側プレビューが同じゲームを保持することを確認する。起動履歴による並び替えでも同様に確認する。
4. 検索中のEscapeは検索だけを解除し、もう一度Escapeを押すと閉じることを確認する。ショートカットの `Alt+1` を再入力した場合は検索状態に関係なく閉じることも確認する。
5. 760×520と620×520のダーク/ライトテーマで横溢れがなく、通常幅ではプレビュー、狭幅では一覧が欠けずに表示されることを確認する。起動失敗、管理画面を開く失敗、再スキャンのエラーも画面内へ表示されることを確認する。

## 8. ファイルシェル
1. Windows Explorer から画面端のハンドルへ複数のファイルとフォルダをドラッグし、ドラッグ中にハンドルまたはシェルフの枠と「ここにドロップしてファイルを追加」の案内が強調されること、ドロップ後に1つのスタックとして追加されることを確認。
2. `Alt+3` と画面端ハンドルの両方で展開・折りたたみでき、設定した左右の端と上・中央・下の位置へ表示されることを確認。「ポインター付近」では、複数モニターを含め現在のポインターがある画面の左右端へ、棚の中央がポインターに近い縦位置で表示され、画面上下からはみ出さないことも確認する。ショートカットまたはクリックで手動展開した棚はポインターが外れても閉じず、ドラッグで一時展開した棚だけがポインター離脱後に自動で折りたたまれることも確認する。
3. シェルフの行中央または右端グリップから Explorer へ取り出せ、6px未満の短いポインター移動は通常の選択として扱われることを確認する。通常はコピー、Shiftを押しながら取り出すと移動になり、ドロップ操作後も項目は自動で消えず、「棚に残す」では維持、「棚から外す」の明示確認後だけ外れることを確認する。ドラッグをキャンセルした場合は確認を出さず棚に残ることも確認する。
4. 取り出し・削除後の「元に戻す」と `Ctrl+Z`（macOSは `Command+Z`）が10秒以内に機能し、アプリ再起動後も残りの棚項目が維持されることを確認。項目を固定すると棚の先頭へ移動し、取り出し・Delete・「固定以外を外す」後も棚に残ること、固定解除後は通常どおり外せることも確認する。
5. 画像（PNG/JPEG/GIF/WebP、25MB以下）、URL、文章を `Ctrl+V` で追加し、開く・コピー・Explorerで表示が機能することを確認。`mailto:`と`tel:`はURLとしてメール／電話の既定アプリで開き、`javascript:`など未許可のスキームは文章として安全に保持されることも確認する。棚のグローバルショートカットを550ms以内に2回押した場合も、現在のクリップボードにある文章・URL・画像が手動項目として追加され、棚が展開して結果またはエラーを表示することを確認する。同じ文章・URLは重複せず、履歴にあった内容は手動項目へ昇格して「クリップボード履歴だけを消去」後も残ることを確認する。
6. 棚から複数回に分けて項目を外し、グローバルショートカットを800ms以上長押しするか、ヘッダーの「最近外した項目を戻す」を押すと、直近の削除バッチが棚の先頭へ戻ることを確認する。繰り返すと24時間以内の削除を新しい順に1バッチずつ戻せ、削除後に追加・固定・改名した他の項目は失われないこと、候補がない場合は棚内にエラーが出ることも確認する。
7. 元ファイルをExplorer側で削除した場合に「見つかりません」と表示され、ファイルシェルを無効にすると `Alt+3` が解除されてウィンドウが消えることを確認。
8. クリップボード履歴が初期状態では無効であることを確認。有効化前からクリップボードにある文章は追加されず、有効化後に新しくコピーした64KB以下の文章とURLだけが「履歴」として追加されることを確認。
9. 「除外するアプリ」に主要なパスワード管理アプリが初期登録されていることを確認する。`C:\Windows\explorer.exe` を追加してExplorerからハンドルへドラッグすると棚が自動展開せず、`Alt+3`による手動表示とハンドルへの明示的なドロップは引き続き使えることを確認する。さらにNotepadを追加し、履歴を有効にしてNotepadで新しい文章をコピーしても自動追加されず、除外を外して別の文章をコピーすると履歴へ追加されること、2連打による明示的な取り込みは除外中も機能することを確認する。
10. 同じ文章をもう一度コピーすると重複せず先頭へ移動し、設定した保存件数を超えても手動追加した項目は残ることを確認。「クリップボード履歴だけを消去」とUndoも確認。
11. `Ctrl+F`（macOSは `Command+F`）または `/` で棚の検索へ移動し、名前・パス・URL・文章を絞り込めることを確認。上下キーで1件、PageUp/PageDownで5件、Home/Endで候補を移動し、Enterで開く、Spaceで選択、`Ctrl+C`（macOSは `Command+C`）でコピーできることを確認する。複数選択時は各項目が改行区切りでコピーされることも確認する。Escapeは検索、選択、棚の順に解除することも確認する。
12. 360×520のダーク/ライトテーマで、検索中と選択アクション表示中も項目が横に欠けず、キーボードカーソルが一覧内へ自動スクロールされることを確認する。ブラウザ確認では `http://localhost:1420/?label=fileShelf&mockClipboardHistory=https%3A%2F%2Fexample.com` を開き、履歴表示と履歴だけの消去も確認する。
13. 画像、文章、URL、通常ファイルを選択し、目のボタンまたは `Q` でクイックプレビューを開けることを確認する。画像は棚の項目IDを通じて安全に読み込まれ、文章とURLは内容、通常ファイルは名前とパスが表示されること、`Q` またはEscapeで棚を閉じずプレビューだけを閉じられることも確認する。ドラッグで一時展開した棚では、ピンボタンまたは `P` でプレビューを固定するとポインターが外れても棚とプレビューが残り、固定解除後は再び自動で折りたたまれることを確認する。プレビューを閉じて開き直した場合は未固定へ戻ることも確認する。
14. 項目を選択して鉛筆ボタンまたは `F2` から棚での表示名を変更でき、Enterまたは「保存」で確定、Escapeで破棄できることを確認する。空白だけ、改行、120文字超の名前が拒否され、表示名を変えてもExplorer上の元ファイル名とパスが変わらないことも確認する。

## 9. クイックキャプチャー
1. `Alt+2` でオーバーレイを開き、本文とタグを入力して350ms程度で「保存済み」になることを確認する。「コピー」で本文を他アプリへ渡せ、閉じて再度開いた場合も下書きが復元されることを確認する。
2. `Ctrl+Enter`（macOSは `Command+Enter`）または「メモに保存」で下書きを保存済みメモへ変換し、本文欄が空の新しい下書きへ戻ることを確認する。
3. 本文でMarkdownを書き、行末のEnterで箇条書き・チェックリスト・番号付きリスト・引用が継続されることを確認する。複数行を選択して`Tab` / `Shift+Tab`でインデント／アウトデントできること、書式ツールバーの見出し2・チェックリスト・箇条書き・引用で構造を切り替えられることを確認する。
4. 「テンプレート」からタスク・会議メモ・アイデア・日報を選び、本文がカーソル位置へ挿入され、テンプレートタグが既存タグへ重複なく追加され、挿入後に本文へフォーカスが戻ることを確認する。プレビュー中または保存中はテンプレート操作が無効になることも確認する。
5. `Ctrl+F`（macOSは `Command+F`）または本文以外で `/` を押し、保存済みメモ検索へ移動することを確認する。通常の「すべて」一覧には未アーカイブのメモだけが表示され、アーカイブ済みメモは「アーカイブ」フィルターまたは`is:archived`で表示できることを確認する。「更新順」「作成順」「タイトル順」で一覧を並べ替えられ、検索中は件数・関連度順表示・一致箇所の強調・本文抜粋が更新されることを確認する。上下キーで1件、PageUp/PageDownで5件、Home/Endで候補を移動し、Enterで選択したメモを開けることを確認する。検索欄で`tag:タグ名`、`is:pinned`、`is:archived`、`has:attachment`と通常の語句を組み合わせ、添付あり・アーカイブ済みフィルターでも絞り込めることを確認する。`Ctrl+K`（macOSは `Command+K`）またはヘッダーの「コマンド」からパレットを開き、操作名を検索してEnterで実行できることも確認する。
6. 保存済みメモを開き、「複製」または `Ctrl+Shift+D`（macOSは `Command+Shift+D`）で本文・タグを新しいメモとして複製でき、編集中の最新内容が失われないことを確認する。「ピン留め」または `Ctrl+Shift+P`（macOSは `Command+Shift+P`）で固定状態を切り替え、一覧の「ピン留め」フィルターで固定したメモだけを表示し、「すべて」で一覧へ戻れることを確認する。メモ一覧上部の「＋」または `Ctrl+N`（macOSは `Command+N`）は常に表示され、保存済みメモからは下書きへ移動し、内容のある下書きからは現在内容を保存して空の新しいメモを開けることも確認する。複製失敗時は元メモを保持し、「複製を再試行」から復帰できることも確認する。
7. 検索欄にフォーカス中のEscapeは、まず絞り込みを解除（絞り込みがなければ検索欄を抜ける）し、もう一度Escapeを押すとオーバーレイが閉じることを確認する。
8. 760×540と680×520のダーク/ライトテーマで、編集欄、タグ、保存操作、書式・テンプレートメニュー、検索、メモ一覧が欠けず、キーボード選択中のメモがスクロール領域内に表示されることを確認する。ブラウザ確認では600×500の狭幅でも同じ項目が欠けないことを確認する。
9. 保存失敗時に入力内容と現在のメモが失われず、「再試行」から保存を再実行できることを確認する。
10. メモ削除とバックアップ復元でMint内の確認ダイアログが表示され、初期フォーカス、Tab循環、Escape、背景クリック、処理中の閉鎖防止が機能することを確認する。復元ではファイル選択後に現在の下書きとメモ件数が示され、失敗後もダイアログ内から再試行できることを確認する。
11. 保存済みメモを開き、OSから複数のファイルを編集欄へドラッグする。ドロップ中に「ここにドロップしてファイルを添付」が表示され、ドロップ後に添付件数とファイル名が更新されること、同じファイルを複数回含めても重複添付されないことを確認する。下書きへドロップした場合は、保存済みメモを選択する必要がある旨が表示され、本文が失われないことも確認する。
12. 未固定のクイックキャプチャーから別アプリへフォーカスを移すと、最新の編集を保存してウィンドウが閉じることを確認する。「ウィンドウを固定」を有効にすると別アプリを操作しても表示が維持され、閉じて開き直してもアプリ終了までは固定状態が残ること、固定中でも閉じるボタン、Escape、`Alt+2` では閉じられることを確認する。固定を解除すると次のフォーカス喪失で閉じ、アプリ再起動後は未固定へ戻ることも確認する。添付、Markdown書き出し、バックアップ入出力のOSファイルダイアログを開いた場合は未固定でもクイックキャプチャーが閉じないことを確認する。
13. 保存済みメモの「アーカイブ」を押すと本文・タグ・更新日時を保ったまま整理でき、「アーカイブ解除」で戻せることを確認する。編集中の`Ctrl+Shift+A`（macOSは `Command+Shift+A`）でも切り替わり、一覧のアーカイブフィルターと検索演算子`is:archived`の結果が一致することを確認する。アプリ再起動後も状態が保持され、バックアップ書き出し・復元でもアーカイブ状態が保たれることを確認する。旧データを含む既存プロファイルを起動してもメモが消えず、未アーカイブとして表示されることを確認する。
14. コマンドパレットで、検索、プレビュー切り替え、保存、テンプレート、コピー、Markdown書き出し、添付、ピン留め、アーカイブ、複製、削除、バックアップ操作が現在の状態に応じて表示されることを確認する。矢印キー・Home/End・Enter・Escape・クリック外しで操作でき、処理中や未選択状態では実行できない操作が表示されないこと、ダーク/ライトテーマと600px以下の幅でも一覧が欠けないことを確認する。

## 11. プラットフォーム固有
- Windows実機にて、アプリの多重起動が防止されているか（Single Instance）。
- トレイアイコンが Windows 11 のタスクバーで正しく表示されるか。
- Windowsの自動起動は `npm run tauri -- build` で作成したリリース版をインストールして確認する。`npm run tauri -- dev` は `localhost:1420` のViteサーバーを使うため、自動起動には登録されない。
