# 手動検証手順 (Manual Verification)

Mint アプリケーションのE2Eテスト環境が構築されるまでの間、リリース前や大幅な機能改修後には以下の手動検証を実施してください。

## 1. アプリケーション起動
1. `npm run tauri -- dev` を実行。
2. アプリケーションのウィンドウが正常に表示されることを確認。
3. デベロッパーツールを開き、コンソールにエラーが出ていないことを確認。

## 2. 機能管理ダッシュボードの設定保存と永続化
1. 機能管理ダッシュボード（設定画面）で「テーマ」を切り替える（Dark / Light）。
2. アプリを完全に終了し、再起動。
3. 変更したテーマが維持されていることを確認（`settings.json` への永続化テスト）。

## 3. ショートカットとトレイメニュー
1. 時計機能のショートカットキー（デフォルト等）を変更し保存。
2. 変更したショートカットキーを入力し、時計オーバーレイが表示・非表示されることを確認。
3. トレイアイコンを右クリックし、メニューから「ダッシュボードを開く」「終了」などの操作が動作することを確認。

## 4. ウィンドウとオーバーレイ
1. 時計オーバーレイが指定秒数後に自動非表示になるか確認。
2. Discord風の「×ボタンで最小化（トレイへ）」が機能し、プロセスがキルされないことを確認。

## 5. Voice to Text
1. Voice to Text のダッシュボード設定でAPIキー、Base URL、モデル名、言語、ショートカットを入力し保存。
2. APIキーがOSのキーリングに保存され、その他の設定がファイルに保存されることを確認（エラーが出ないこと）。
3. 機能を有効にして音声ファイルパスを入力し、「文字起こしを実行」で文字起こし結果が表示されることを確認。
4. APIキー未設定、無効な音声ファイルパス、APIエラー時に画面上へエラーが表示されることを確認。
5. Voice to Text を無効にした場合、設定したショートカットキーがOSのグローバルショートカットから解除されることを確認。

## 6. アップデーター
1. `npm run dev` を実行し、`http://localhost:1420/?mockUpdate=available` を開く。
2. 一般設定の「アップデート」に、現在・更新後のバージョン、公開日、リリースノートが表示されることを確認。
3. 「ダウンロードして再起動」を押し、ダウンロード進捗、インストール中、再起動中の順に表示が変わることを確認。
4. `http://localhost:1420/` では「更新を確認」を押すと「最新バージョンです」と表示されることを確認。
5. 実リリース前にはテスト用の旧バージョンをインストールし、公開済みの署名付き Release へ更新できることを対象OSで確認。

## 7. プラットフォーム固有
- Windows実機にて、アプリの多重起動が防止されているか（Single Instance）。
- トレイアイコンが Windows 11 のタスクバーで正しく表示されるか。
